### [FeaturedFilmCard] Layout & Sizing

**Files read:** FeaturedFilmCard.jsx, index.css (lines 441-461), films.js, tailwind.config.js

**Findings:**

- CRITICAL: Video `maxWidth: var(--video-width)` applies at ALL breakpoints. On mobile (flex-col), videos render at 42-58% container width instead of full-width. The media query in index.css protects padding/gap/text-padding but NOT the video width.

- HIGH: Three-layer styling conflict — The same layout is controlled by: (a) Tailwind classes in JSX (`gap-6`, `pt-0`, `w-full`), (b) CSS custom properties as inline styles (`--video-width`, `--card-gap`, `--text-padding-top`), (c) descendant selectors in index.css (`.film-card > div > div:last-child`). This split makes layout unpredictable and hard to debug.

- HIGH: `pt-0` Tailwind class on text div (line 70) conflicts with CSS `var(--text-padding-top)` applied via `.film-card > div > div:last-child` (index.css:458). CSS selector specificity (0-1-2) beats Tailwind utility (0-1-0), so `pt-0` is silently ignored on desktop. Misleading.

- HIGH: Video sizing uses `w-full` + `maxWidth` + `flex-shrink-0` instead of explicit flex-basis or width. Flex layout computes available space ambiguously — the text column `flex-1` gets leftover space, but "leftover" depends on how the browser resolves the video's intrinsic vs constrained width.

- MEDIUM: LAYOUT_VARIANTS uses arbitrary magic numbers with no formula. paddingTop: 8.75-13.75rem, textPaddingTop: 2.5-7.5rem, videoWidth: 42-58%. Creates unpredictable feel across cards.

- MEDIUM: Gap has dual sources — `gap-6` (24px) in JSX for mobile, `var(--card-gap)` via CSS for desktop. Two separate mechanisms for the same property at different breakpoints.

- LOW: `pt-0` class is dead code on desktop (overridden) and redundant on mobile (default is 0).

- LOW: `lg:px-0` removes horizontal padding on desktop. Video + text sit edge-to-edge in the container, which combined with variable widths affects perceived text position.

**Doc accuracy:** No MDD feature doc exists for film cards.

### [04] Featured Card — Scroll-Tied Fade
**Files read:** `src/components/films/FeaturedFilmCard.jsx`, `src/context/SmoothScrollContext.jsx` (444 LOC).

**Resolved / verified:**
- ✅ `RAMP = window.innerHeight * 0.20` matches doc.
- ✅ Smootherstep formula `c³(c(c·6−15)+10)` is correct.
- ✅ Reduced-motion short-circuits before listener registration.
- ✅ `addScrollListener` cleanup on unmount.
- ✅ Resize handler refreshes `RAMP` and re-runs `update()`.
- ✅ Initial `update()` call runs once at mount so cards already in view aren't transparent.
- ✅ Fallback `unsubscribe()` returned from `addScrollListener` cleans up via `scrollListenersRef.current.delete(listener)`.

**Open / new findings:**
- 🟡 **M04-1 — Wrapperref scope mismatches doc.** Doc 04 lines 38-43 say opacity is "applied to the video **wrapper div** (`FeaturedFilmCard.jsx:118-129`) rather than the `<video>` element, so the poster `backgroundImage` set on the wrapper fades together with the playing video frame — they appear as one block." But the actual `wrapperRef` in `FeaturedFilmCard.jsx:51, 134` sits on the *outer `<article>`* — meaning the **entire card** (mobile-only title above the video, video, metadata line, description paragraph) fades, not just the video block. Either the doc is stale (the article-level fade is intentional, in which case update doc) or the ref is mis-targeted (regression — should be on the inner video div at line 168-178). Visually inspect to confirm intent; if unintentional, move `wrapperRef` to the inner video div.
- 🟡 **M04-2 — `SmoothScrollContext.jsx` is 444 LOC.** Exceeds the 300-line gate. Candidate split: pull the wheel/keyboard/touch handlers into a `useSmoothScrollHandlers` hook, or split the desktop-vs-native branches into separate effects.
- 🟢 **L04-1 — `addScrollListener` deps churn.** `useEffect[addScrollListener]` re-runs whenever the context value re-creates the function. `addScrollListener` is `useCallback([])` so stable, but worth confirming on real-world usage that no parent re-renders cause the effect to re-init (cheap to verify; cheap to migrate to a ref-based subscription pattern if needed).
- 🟢 **L04-2 — On mobile, `setScrollY(window.scrollY)` is called per native scroll event without throttling** (line 358-359). Combined with `passive: true`, the cost is low, but every scroll event triggers a React re-render of every consumer of `scrollY`. The desktop path throttles to 15fps (line 107). Mirror the throttle on mobile.

**Test coverage:** No new test (doc explicitly defers — "too visual/timing-sensitive"). `tests/homepage.spec.js` provides minimal regression coverage.
**Doc accuracy:** ⚠ minor drift — see M04-1.

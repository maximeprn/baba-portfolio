## Findings by Severity

### CRITICAL (1)

1. **Photo project images use `/src/assets/` paths — broken in production**
   - *Where:* `src/data/photoProjects.js` — all `coverImage` and `photos[].src` paths
   - *Impact:* Every photo on the Photos page, photo project galleries, and FloatingGalleryHero will show broken images in production
   - *Why:* Vite doesn't serve `/src/assets/` at runtime. Files must either be in `public/` (referenced as `/photos/...`) or imported via ES modules
   - *Fix:* Move photos to `public/photos/` and update paths, OR use `import` statements
   - *Effort:* 10 min (path updates) + file moves

### HIGH (3)

2. **ShowreelOverlay body overflow conflict with SmoothScrollContext**
   - *Where:* `src/pages/Films.jsx:15-19`
   - *Impact:* Opening/closing showreel may break smooth scroll state on desktop
   - *Fix:* Save/restore `previousOverflow` pattern (same as Lightbox)
   - *Effort:* 2 min

3. **FilmModal body overflow conflict with SmoothScrollContext**
   - *Where:* `src/components/films/FilmModal.jsx:11-17`
   - *Impact:* Same issue as #2 — closing modal sets `overflow = ''` instead of restoring previous value
   - *Fix:* Save/restore pattern
   - *Effort:* 2 min

4. **SmoothScrollContext exceeds 300-line quality gate (415 lines)**
   - *Where:* `src/context/SmoothScrollContext.jsx`
   - *Impact:* Maintainability concern — complex scroll logic, touch handling, keyboard nav all in one file
   - *Fix:* Extract touch and keyboard handlers into separate hooks
   - *Effort:* 15 min

### MEDIUM (5)

5. **Mobile hero uses `<a href>` instead of React Router `<Link>`**
   - *Where:* `src/components/ui/HeroSection.jsx:37-44`
   - *Impact:* Navigation links cause full page reload on mobile instead of client-side routing
   - *Fix:* Replace `<a href>` with `<Link to>`
   - *Effort:* 2 min

6. **Duplicate overlay pattern — ShowreelOverlay and FilmModal**
   - *Where:* `src/pages/Films.jsx:11-53` and `src/components/films/FilmModal.jsx`
   - *Impact:* Code duplication — both implement close button, escape key, spinner, iframe
   - *Fix:* Extract shared `VimeoOverlay` component
   - *Effort:* 10 min

7. **CLAUDE.md architecture section is stale**
   - *Where:* `CLAUDE.md` — Architecture section lists deleted files (IntroSection, galleryLayout, useStaggeredText, useParallaxLayers, FilmDetail) and wrong fonts
   - *Impact:* AI and collaborators get incorrect context about the project
   - *Fix:* Update to reflect current file structure and font names
   - *Effort:* 5 min

8. **FilmCard has no mobile/responsive handling**
   - *Where:* `src/components/films/FilmCard.jsx`
   - *Impact:* `aspect-[2/1]` with absolute positioned images may break on small screens
   - *Fix:* Add responsive breakpoints or mobile layout path
   - *Effort:* 15 min

9. **Inline `fontFamily` instead of Tailwind classes in FeaturedFilmCard/FilmCard**
   - *Where:* `src/components/films/FeaturedFilmCard.jsx:90,98,100,114` and `FilmCard.jsx:61,69,74`
   - *Impact:* Inconsistency with design system — harder to maintain/change fonts
   - *Fix:* Replace `style={{ fontFamily: 'Helvetica Now Display' }}` with `className="font-header"`
   - *Effort:* 3 min

### LOW (4)

10. **VideoPlayer.jsx appears unused**
    - Not imported by any current component (films use FilmModal with direct iframe)
    - *Fix:* Verify and remove if truly unused
    - *Effort:* 2 min

11. **Two `<h1>` elements in mobile hero**
    - *Where:* `HeroSection.jsx:63-68`
    - *Impact:* Minor SEO concern — multiple h1s on same page
    - *Fix:* Wrap both in single `<h1>` with spans
    - *Effort:* 2 min

12. **PersistentHeroText assumes exactly 3-word tagline**
    - *Where:* `PersistentHeroText.jsx:15`
    - *Impact:* Changing tagline in siteConfig could break layout
    - *Fix:* Add bounds checking or restructure
    - *Effort:* 3 min

13. **Unused font files (Helvetica Now Micro) in public/fonts**
    - No @font-face declarations reference them
    - *Fix:* Remove if not planned for future use
    - *Effort:* 1 min

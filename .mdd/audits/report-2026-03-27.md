# Film Cards Audit Report — 2026-03-27

## Executive Summary

The FeaturedFilmCard component has unpredictable text/video sizing because its layout is split across three styling mechanisms (Tailwind classes, CSS custom properties, and descendant selectors in index.css) that conflict with each other. One critical bug makes videos render at partial width on mobile. The regular FilmCard is clean by comparison.

## Findings

### CRITICAL

**F1. Video constrained to 42-58% width on mobile**
- `maxWidth: var(--video-width)` is set as inline style at all breakpoints
- On mobile (flex-col), videos render at 42-58% of container width instead of full-width
- The `.film-card` media query in index.css only protects padding/gap/text-padding, NOT video width
- **File:** FeaturedFilmCard.jsx:48-53, index.css:446-461
- **Fix:** Apply `--video-width` maxWidth only on desktop via CSS media query or conditional inline style

### HIGH

**F2. Three-layer styling makes layout unpredictable**
- Same elements controlled by: (a) Tailwind utilities in JSX, (b) CSS custom properties as inline styles, (c) descendant selectors in index.css with media queries
- To understand any element's actual style, you must check all three sources and reason about specificity
- **File:** FeaturedFilmCard.jsx + index.css:441-461
- **Fix:** Consolidate to one approach — use CSS custom properties in index.css for all variant-driven values, remove conflicting Tailwind classes

**F3. `pt-0` class silently overridden by CSS**
- Text div has `pt-0` (padding-top: 0) in JSX
- `.film-card > div > div:last-child { padding-top: var(--text-padding-top) }` in index.css has higher specificity (0-1-2 vs 0-1-0)
- `pt-0` is dead code on desktop — creates false impression of zero top padding
- **File:** FeaturedFilmCard.jsx:70, index.css:458
- **Fix:** Remove `pt-0`, let CSS rule be the single source of truth

**F4. Video column uses ambiguous flex sizing**
- `w-full` + `flex-shrink-0` + `maxWidth` constraint instead of explicit width or flex-basis
- Flex space distribution depends on how the browser resolves constrained width, creating browser-dependent behavior
- **File:** FeaturedFilmCard.jsx:46-47
- **Fix:** Use explicit width on desktop (matching the variant's videoWidth), `w-full` only on mobile

### MEDIUM

**F5. Arbitrary LAYOUT_VARIANTS values**
- Four hardcoded variants with seemingly random values (paddingTop: 8.75-13.75rem, textPaddingTop: 2.5-7.5rem, videoWidth: 42-58%)
- No formula or design system relationship between values
- Creates the "unpredictable" feel the user described
- **File:** FeaturedFilmCard.jsx:6-11
- **Fix:** Reduce to 2 clean variants (e.g., video-large 58% and video-small 45%) with proportional text offsets

**F6. Gap defined in two places for two breakpoints**
- Mobile: `gap-6` Tailwind class in JSX (24px)
- Desktop: `var(--card-gap)` via CSS media query (2-3.5rem)
- **File:** FeaturedFilmCard.jsx:40, index.css:453
- **Fix:** Define gap entirely through CSS custom property with a mobile default

### LOW

**F7. FilmCard: hardcoded max-w-[584px] on description/credits only**
- Description and credits capped at 584px, but title is uncapped — creates inconsistent text block width
- **File:** FilmCard.jsx:42, 49

**F8. Unnecessary flex wrapper on FilmCard header**
- `inline-flex items-center justify-center` around a single h3 — adds complexity with no effect
- **File:** FilmCard.jsx:33

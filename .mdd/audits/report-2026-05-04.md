## NEW — HIGH (2)

### N1. Slideshow batch-preloads all 116 photos at mount
**File:** `src/components/photos/FloatingGalleryHero.jsx:128-132`
**Impact:** ~116 simultaneous network requests when a visitor lands on `/photos`. Heavy on mobile networks; defeats the cascade-load pattern used elsewhere. Spec at `.mdd/docs/01-photos-hero-slideshow.md:35` says "preload the next 3" — this is the opposite.
**Fix:** Maintain a rolling 3-ahead preload window inside `advance()`.

### N2. `isMobile` captured once at hook mount
**File:** `src/components/photos/FloatingGalleryHero.jsx:82`
**Impact:** Initial `window.innerWidth` read is closed over by the cycling effect; resizing across the 768 breakpoint (rotation, devtools) does not update the landscape-skip filter.
**Fix:** Use `matchMedia('(max-width: 767px)').addEventListener('change', …)` or include `isMobile` in the effect dependency array.

## NEW — MEDIUM (4)

### N3. `FloatingGalleryHero.jsx` is 372 LOC — exceeds 300-line gate
**File:** `src/components/photos/FloatingGalleryHero.jsx`
**Fix:** Extract `usePhotoSlideshow` hook and `NameOverlay`/`MobileHeroNameOverlay` into separate files.

### N4. Reduced-motion path never computes brightness
**File:** `src/components/photos/FloatingGalleryHero.jsx:124-125`
**Impact:** When `prefers-reduced-motion` is on, the effect returns early; `isLight` stays `false`, so the name overlay uses the light-mode color regardless of the actual photo. The static-photo fallback works visually but with wrong contrast on bright shots.
**Fix:** Run `preloadImage(initial[0])` once to set brightness even when cycling is disabled.

### N5. Mobile name overlay color is fixed (white)
**File:** `src/components/photos/FloatingGalleryHero.jsx:336`
**Impact:** Desktop has adaptive black/white based on photo brightness; mobile is always white. Accessibility regression on bright photos.
**Fix:** Thread `currentPhoto.isLight` into the mobile overlay and switch color.

### N6. `advance()` silently fails after 20 retries
**File:** `src/components/photos/FloatingGalleryHero.jsx:104-121`
**Impact:** On mobile, if 20 consecutive shuffled photos are landscape, `advance` returns without setting state — user sees a frozen frame for one cycle. Rare, invisible failure.
**Fix:** Pre-filter `heroPhotos` for portrait once on mobile rather than reject-and-retry.

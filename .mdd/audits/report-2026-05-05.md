## NEW — HIGH (2)

### H02-1. `siteConfig.seo.ogImage` points at a non-existent asset
**File:** `src/data/siteConfig.js:131` → `/images/og-image.jpg`
**Impact:** `public/images/` does not exist. Social shares (Slack, iMessage, X, Discord) will fail to render an OG preview. The string is wired into `Helmet` indirectly via siteConfig but only specific pages override og tags (Films does, About/Photos do not include `og:image`). Production-blocker for marketing.
**Fix:** Add a 1200×630 OG asset under `public/og/og-image.jpg` (or similar) and update siteConfig path; OR null out ogImage and let consumers fall back to a default.

### H02-2. Design-system doc 02 still describes a wordmark hero that no longer exists
**Files:** `.mdd/docs/02-design-system.md:80, 88-90, 171, 226-227` ↔ `src/components/ui/PersistentHeroText.jsx` (orphan), `src/components/ui/HeroSection.jsx`
**Impact:** Doc 02 says the 140px **BASILE DESCHAMPS** rendered by `PersistentHeroText` "IS the logo" and is "the canonical brand expression." But after commit `0e6bcbb` ("Redesign hero: full-bleed media + scrolling nav + bio/contact overlay"):
- `PersistentHeroText.jsx` is no longer imported by any consumer.
- `HeroSection` and `FloatingGalleryHero` both render `<HeroBioOverlay />` instead — bio + selected clients + contact info.
- `--text-hero` (140px) is defined in CSS but unused.
A new contributor (or AI agent) would design to a brand identity that doesn't exist on the live site.
**Fix:** Two parts. (1) Update doc 02 — replace *Typography > The wordmark*, *Components > UI primitives* (`PersistentHeroText` line), and *Assets > Logo / wordmark* with the new bio overlay description. Add `HeroBioOverlay` as the canonical hero expression. (2) Decide on `PersistentHeroText.jsx`: delete + remove `--text-hero` token, OR explicitly mark it as a retired component in the doc.

## NEW — MEDIUM (6)

### M01-1. Doc 01 (slideshow) has multiple stale claims
**File:** `.mdd/docs/01-photos-hero-slideshow.md`

- Doc says "preload the next 3"; code uses `PRELOAD_AHEAD = 10`.
- Frontmatter `known_issues` lists N3, N4, N5, N6 — but N3/N4/N5 are no longer applicable.
**Fix:** Rewrite Architecture, Business Rules, Known Issues. Bump `PRELOAD_AHEAD` mention to 10. Remove brightness/MobileHeroNameOverlay paragraphs.

### M03-1. `CollapsedFilmCard.jsx` exceeds the 300-LOC gate (377)
**File:** `src/components/films/CollapsedFilmCard.jsx`
**Impact:** Codified rule violation (design-system doc 02 line 209).
**Fix:** Extract overlay + close-indicator subcomponents, or pull open/close orchestration into a `useShutterAnimation` hook. Roughly 80-line reduction is achievable without behavior change.

### M03-2. Doc 03 hover behavior is inverted from code
**Files:** `.mdd/docs/03-collapsed-film-cards.md` (Hover affordances section) ↔ `CollapsedFilmCard.jsx:289, 294, 302`
**Impact:** Doc says "the entire band inverts to a black bar with white text"; code applies `group-hover/row:bg-gray-900 group-hover/row:text-white` to **each** of the three text spans, producing three separate per-chip pills. Cosmetic, but the spec is the source of truth and a reviewer comparing the two will be confused.
**Fix:** Update the "Hover affordances" table in doc 03 to describe per-chip pills. Add a note that the trigger is on `group/row` (outer) but the visual is per-text-element.

### M04-1. Doc 04 says the fade is on the video wrapper; code applies it to the whole article
**Files:** `.mdd/docs/04-featured-card-fade-on-scroll.md:38-43` ↔ `FeaturedFilmCard.jsx:51, 134`
**Impact:** Doc explicitly says opacity is applied to the inner video wrapper (lines 118-129 in the current code, the `<div onClick={handleClick}>` containing the `<video>`). But `wrapperRef` is on the outer `<article>` — so on mobile the title above the video and on desktop the description text all fade together with the video. The visual likely still looks fine because the fade only kicks in within ~20% of vh of the edge, but the doc is wrong about *what* fades.
**Fix:** Inspect visually. If intentional (article-wide fade reads as one motion), update doc 04 to reflect that. If unintentional, move `wrapperRef` to the inner video div at lines 168-178.

### M04-2. `SmoothScrollContext.jsx` exceeds the 300-LOC gate (444)
**File:** `src/context/SmoothScrollContext.jsx`
**Impact:** Same rule violation as M03-1, larger file. Carry-over (was implicit in 2026-05-04, never explicitly flagged).
**Fix:** Pull wheel/keyboard/touch handlers into a `useSmoothScrollHandlers` hook, or split desktop-vs-native branches into separate effects. Aim for ~280-LOC main file.

### M05-4. (carry-over) `films.js` — 10 of 15 films have `thumbnail: null`
**File:** `src/data/films.js`
**Impact:** Featured cards render `backgroundImage: undefined` when thumbnail is null, so on slow connections users see a black box until the video loads. UX regression on first-impression loading. Demoted from MEDIUM in prior audits but still present.
**Fix:** Generate poster stills for all 15 films (FFmpeg `-ss 00:00:01 -frames:v 1`), or render a global "title-on-black" fallback poster.

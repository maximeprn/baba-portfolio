# MDD Audit Report — Sanity / CMS layer

**Date:** 2026-05-21
**Scope:** `/mdd audit the Sanity part` — docs 07–14 + every Sanity-touching
source file (schemas, Studio config/tools, `src/sanity/*`, all `cms:*` +
migration scripts, build wiring).
**Notes:** [.mdd/audits/notes-2026-05-21.md](notes-2026-05-21.md)

## 1. Executive summary

The Sanity **runtime path** — schemas → `fetch-cms-content.mjs` → `cms.json` →
`loader.js` → components — is healthy and internally consistent. The committed
`cms.json` snapshot proves it: 17 films, 22 photo projects, 115 hero photos,
all in shapes that match the current schemas. The site builds and renders.

Every finding is **drift**, not breakage. Two schema rewrites happened —
(a) the Mux video migration (doc 12) and the removal of `featured`/
`imagePosition` from `film`, (b) the hero-overlay move to fixed insets
(docs 13/14) — and the **migration scripts and MDD docs were not updated to
match**. The result: one migration script (`upload-films.mjs`) now writes 4
fields the schema has dropped, one reset script (`seed-cms.mjs`) seeds stale
overlay data, the live-reload watcher misses 3 of 6 content types, and docs
07/09/10/12 describe fields and behaviours that no longer exist.

**13 findings: 0 Critical, 2 High, 7 Medium, 4 Low.**

## 3. Findings by severity

### 🔴 HIGH

**H1 — `upload-films.mjs` writes 4 fields the `film` schema no longer has.**
`scripts/upload-films.mjs` builds film docs with `featured` (L103),
`videoType` (L106), `videoFile` (L107) and `imagePosition` (L113). The current
`film` schema declares none of them (Mux migration dropped `videoFile`/
`videoType`; `featured` + `imagePosition` were removed when the homepage moved
to row-index alternation). `cms:upload-films` is a documented, supported
re-import path. Running it today creates docs with 4 orphan "unknown field"
warnings in Studio and never writes the new `visible` field.
*Fix:* drop the 4 orphan fields, add `visible`. ~5 min.

**H2 — Doc 10 (`10-cms-films.md`) is substantially out of date.**
Data Model + Business Rules still document `featured`, `videoType`,
`videoFile`, `imagePosition`; omit `videoMux` + `visible`; Business Rule 1
describes a featured/collapsed interplay that no longer exists; build-plan
step 3 names `getFeaturedFilms()` (loader actually exports
`getNonCollapsedFilms()`); `status` says "awaiting user-side run" (17 films are
live); `known_issues` claims ESLint config is missing (`.eslintrc.cjs` exists).
*Fix:* rewrite doc 10 Data Model/Business Rules/status/known_issues. ~15 min.

### 🟠 MEDIUM

**M1 — "Sanity webhook Delay (~60s)" myth in 3 scripts.**
`seed-cms.mjs` (L127-129), `upload-photo-projects.mjs` (L177-179) and
`fix-photo-project-ranks.mjs` (L16) all tell the reader to set a webhook
"Delay" to coalesce deploys. CLAUDE.md states plainly: "Sanity does NOT have a
built-in webhook debounce." `upload-photo-projects.mjs` even contradicts itself
(its printed reminder at L189-192 says the opposite). Following the bad advice
risks a Vercel deploy-quota burn.
*Fix:* replace the myth with the real workflow (disable auto-webhook → run →
Studio Deploy). ~5 min.

**M2 — Doc 07 hero-overlay model is stale.**
Doc 07 Data Model lists `offsetX`/`offsetY` as live positioning fields and
`stackRowGap` (default 24), plus a 9-anchor enum with center anchors. Reality
(docs 13/14): offsets retired (`hidden:true`), positioning is fixed insets,
6 anchors only. Business Rules 8–9 describe the obsolete offset maths.
*Fix:* add a supersession note + correct the table/rules. ~10 min.

**M3 — Doc 07 webhook GROQ filter omits `film`.**
Business Rule 12's filter is missing `film`. CLAUDE.md's copy is correct.
Pasting doc 07's version into the live webhook would make published film
changes silently never deploy.
*Fix:* add `"film"` to the filter in doc 07. ~1 min.

**M4 — `seed-cms.mjs` seeds stale overlay data.**
The `HERO_OVERLAY` constant uses retired `offsetX`/`offsetY` and omits
`textSize` + `autoShrinkSmallScreens`; `SITE_SETTINGS` omits `navLinkSize`/
`navLinkActiveSize`. `cms:seed` is the documented "reset to fresh" path —
running it reverts the overlay to pre-doc-13/14 data.
*Fix:* update the seed constants to the current field shape. ~10 min.

**M5 — `watch-cms.mjs` live-reload misses 3 of 6 content types.**
The listen query is `*[_id in ["siteSettings","heroOverlay","showreel"]]` —
no `heroPhotos`, `photoProject`, or `film`. Editing those in Studio does not
refetch in local dev. CLAUDE.md advertises `cms:watch` without this caveat.
*Fix:* widen the query to `*[_type in [...all 6...]]`. ~5 min.

**M6 — `src/sanity/client.js` is dead code with a false comment.**
Nothing imports the local module — the fetcher, watcher and every script
construct their own `@sanity/client`. The header claims "Used by the
build-time fetcher."
*Fix:* correct the comment (keep the file as an intentional read-only client
surface, or delete it — flagged for decision). ~3 min.

**M7 — `imagePosition` ghost field on `photoProject`.**
Doc 09 documents `imagePosition`; the schema has no such field; the fetcher
doesn't project it; but `upload-photo-projects.mjs` L138-140 still writes it
(orphan field), and `loader.js` L142 comment still lists it.
*Fix:* remove from doc 09, the script, and the loader comment. ~5 min.

**M8 — Doc 12 status + resolution tier stale.**
Frontmatter `status: PLANNED — implementation not yet started`; the feature is
fully shipped. Doc says "720p" throughout; the `muxInput` config and migration
script use `1080p`.
*Fix:* set status BUILT, fix the tier references. ~5 min.

**M9 — `heroPhotos` validation severity contradicts doc 08.**
Schema uses `.min(1).error(...)` (blocks publish). Doc 08 specifies "warning,
not error — empty arrays are tolerated by the build's fallback path."
`photoProject.photos` uses `.warning()` — inconsistent.
*Fix:* change `heroPhotos` to `.warning()` to match doc + sibling schema. ~2 min.

### 🟡 LOW

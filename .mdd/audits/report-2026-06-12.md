## 1. Executive summary

The **code is healthy**: lint is clean, no new runtime bugs were found in any cluster
(expand/collapse orchestration, FLIP, overlay layout, fade, slideshow, CMS pipeline all
verified line-by-line), and all 13 findings from the 2026-05-21 CMS audit landed
(2 with small residues). The two open bugs in doc 01 remain open and tracked.

The debt is in two places:

1. **The test suite is red on main** — 3 of 56 chromium tests fail deterministically,
   all from test↔code drift, not product bugs: the Mux migration broke the films-CDN
   spec's premise, a Sanity Studio version changed the login DOM under the studio spec,
   and the Lightbox's new swipe-carousel broke the lightbox spec's `img.first()`
   assumption. A 4th (anchored switch) is flaky. "All tests pass before committing"
   has silently stopped being true.

2. **The Other-Projects grid rework + card unification (PRs #29/#31, commit 5b96fd9)
   never got their docs updated.** Docs 03/05/06 describe a retired layout (single-row
   3-zone bands, `cards` breakpoint offsets) and a superseded orchestration model
   (`flushSync` anchored open) — docs 05 and 06 now contradict each other about the
   same shared mechanism. CLAUDE.md still names three components that don't exist.

**34 findings: 0 Critical, 6 High, 16 Medium, 12 Low.** Plus 2 false alarms dismissed
during verification (see notes).

## 3. Findings by severity

### 🔴 CRITICAL — none

### 🟠 HIGH

**H1 — 3 E2E tests deterministically red + 1 flaky (test↔code drift, not prod bugs).**
(a) `cms-films-cdn.spec.js:20` expects an `<img src*="cdn.sanity.io">` on `/`; films now
ship Mux posters (17/17 `muxPosterUrl`, only 5 Sanity thumbnails) as `<video poster>`,
and collapsed bodies are unmounted at rest. (b) `cms-studio.spec.js:20`'s shell selector
matches nothing in the current Sanity v4 login DOM — /admin verified healthy via page
snapshot. (c) `featured-photo-cards.spec.js:130` grabs `img.first()` inside the
Lightbox, which is now a 3-slide carousel — `.first()` is the *prev* slide at x≈-913.
(d) `:201` anchored-switch is flaky under load.
*Fix:* (a) accept Mux poster/CDN as proof-of-CMS; (b) match the login screen or Studio
shell; (c) add a `data-lightbox-slot` attr and target the current slide; (d) observe
after fixes. ~45 min.

**H2 — Doc 06 is the most out-of-date doc in the project.** Title ("single-row"),
Purpose, and the 3-tier Layout section describe a retired band layout; reality is a
`grid-cols-2/3/4` grid of stacked cards with `col-span` breakout (Photos.jsx:166,
CollapsedPhotoCard.jsx:398,427). The "reference implementation / shares the cards
breakpoint" framing is inverted. *Fix:* retitle + rewrite Purpose/Layout. ~30 min.

**H3 — Doc 05's "Anchored Open" + `flushSync` narrative is superseded.** Code runs the
sibling collapse *deferred* via the `photo-card-expand-done` event so open + close
animate in parallel (FeaturedPhotoCard.jsx:154-159, 365-445). Docs 05/06 contradict
each other on the same shared mechanism. *Fix:* rewrite those sections, cross-ref
doc 06's handoff as canonical. ~30 min.

**H4 — CLAUDE.md misleads navigation in 4 places.** Architecture tree lists
nonexistent `FilmCard`, `ProjectCard`, `PhotoGrid` and omits the 5 real card
components; "Fullscreen **sticky** video hero" (sticky dropped, 3d1dec4); Gotcha
"Photo images must be in `public/photos/`" (Sanity CDN now); hero video filename.
Doc 12's source_files also lists the phantom `FilmCard.jsx`. *Fix:* ~10 min.

**H5 — CollapsedPhotoCard + the cross-type single-expand handoff have zero test
coverage.** No spec uses `[data-collapsed-photo-card]`; doc 06's test_files claim is
false. The headline behavior of docs 05+06 (featured↔collapsed handoff) is unguarded.
*Fix:* add a collapsed-photo-cards spec incl. one cross-type test. ~45 min.

**H6 — `package.json` `vercel-build` (`vite build`) diverges from
`vercel.json` `buildCommand` (`bash scripts/vercel-build.sh`).** Running the npm
script directly ships a stale `cms.json` with no CMS fetch. Carried from the last
audit's "noted" list. *Fix:* point the npm script at the shell script. ~2 min.
